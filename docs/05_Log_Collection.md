# Log Collection

## Overview

Effective Security Information and Event Management (SIEM) begins with reliable log collection. In this project, Linux authentication events are continuously collected from the Ubuntu target server and forwarded to Splunk Enterprise using the Splunk Universal Forwarder.

The objective is to centralize authentication logs, enabling real-time monitoring, threat detection, and incident investigation.

---

# Objectives

The log collection pipeline was designed to achieve the following objectives:

- Collect Linux authentication events.
- Forward logs to Splunk Enterprise in near real time.
- Ensure reliable and continuous log ingestion.
- Store events within a dedicated index.
- Support detection engineering and dashboard visualization.

---

# Log Source

The primary data source for this project is the Linux authentication log.

```text
/var/log/auth.log
```

This file records authentication-related activity generated by the OpenSSH service.

---

# Types of Events Collected

The following authentication events are collected and indexed.

- Successful SSH logins
- Failed SSH login attempts
- Invalid usernames
- Incorrect passwords
- Accepted passwords
- Session opened
- Session closed
- SSH daemon activity
- Authentication failures

# Splunk Universal Forwarder

The Splunk Universal Forwarder continuously monitors the authentication log for new events.

Whenever new entries are written to `/var/log/auth.log`, they are immediately forwarded to Splunk Enterprise.

Advantages include:

- Lightweight
- Reliable
- Near real-time forwarding
- Minimal CPU usage
- Secure communication

---

# Forwarding Configuration

The Universal Forwarder was configured to:

- Monitor `/var/log/auth.log`
- Forward events to the Splunk Enterprise server
- Send data to TCP port **9997**
- Tag events with the appropriate source and sourcetype

---

# Challenges Encountered

Several issues were identified during deployment.

## SSH Password Authentication

Password authentication was disabled by default on the Ubuntu cloud image.

The SSH configuration was updated and the service restarted to allow password-based testing.

---

## Forwarder Verification

Connectivity between the Universal Forwarder and Splunk Enterprise was verified before attack simulations were performed.

This ensured all authentication events were captured successfully.

---

# Outcome

The log collection pipeline successfully centralized Linux authentication logs within Splunk Enterprise.

Every authentication event generated during the attack simulations was collected, indexed, and made available for searching, dashboard visualization, and alert generation.

# Splunk Configuration

## Overview

Splunk Enterprise serves as the centralized Security Information and Event Management (SIEM) platform within this lab. It is responsible for receiving, indexing, searching, visualizing, and alerting on Linux authentication events generated by the monitored Ubuntu server.

This document describes the complete Splunk configuration performed to enable centralized log collection and security monitoring.

---

# Objectives

The Splunk configuration was designed to achieve the following objectives:

- Deploy Splunk Enterprise
- Configure a dedicated index for Linux authentication logs
- Enable data receiving from Universal Forwarders
- Configure log forwarding
- Validate successful log ingestion
- Prepare the environment for detection engineering

---

# Splunk Enterprise Installation

Splunk Enterprise was installed on a dedicated Ubuntu EC2 instance hosted in AWS.

The installation included:

- Splunk Enterprise
- Splunk Web Interface
- Search Head
- Indexer

After installation, the Splunk service was started and configured to launch automatically during system startup.

---

# Splunk Web Interface

The Splunk Web Interface provides centralized management for:

- Searching logs
- Creating dashboards
- Building alerts
- Managing indexes
- Configuring data inputs
- User administration

The interface was accessed using:

```

https://<Splunk-Server-IP>:8000

```

---

# Index Configuration

A dedicated index named **linux_auth** was created to store Linux authentication events.

## Purpose

Separating authentication logs into a dedicated index provides several advantages:

- Faster searches
- Better data organization
- Simplified access control
- Improved dashboard performance
- Easier alert management

---

# Data Receiving Configuration

To receive events from Universal Forwarders, Splunk was configured to listen on TCP port **9997**.

## Receiving Port

```

9997/TCP

```

This port receives forwarded authentication logs from monitored Linux systems.

---

# Universal Forwarder Configuration

The Splunk Universal Forwarder was installed on the Ubuntu target server.

Its primary responsibility is to monitor local log files and securely forward new events to the Splunk Enterprise server.

---

# Monitored Log File

The forwarder was configured to continuously monitor:

```

/var/log/auth.log

```

This file contains authentication-related events generated by the OpenSSH service.

---

# Forwarding Destination

Authentication logs are forwarded to the Splunk Enterprise receiver.

Destination:

```

<Splunk Server IP>:9997

```

---

# Log Monitoring

The Universal Forwarder continuously monitors:

- Failed login attempts
- Successful logins
- Invalid usernames
- SSH daemon activity
- Session creation
- Session termination

New events are forwarded to Splunk in near real time.

# Troubleshooting

During deployment, several issues were encountered and resolved.

## Password Authentication Disabled

The Ubuntu cloud image disabled SSH password authentication by default.

The SSH configuration was updated to enable password-based authentication for testing purposes.

After modifying the SSH configuration, the SSH service was restarted and validated.

---

## Forwarder Connectivity

Forwarder connectivity was verified by confirming:

- Receiver availability
- Port accessibility
- Active forward-server connection
- Successful log indexing

---

## Log Collection Verification

Authentication events were validated by generating:

- Successful SSH logins
- Failed SSH logins
- Invalid username attempts

Each event appeared within Splunk shortly after generation.
